## 🔨 Key Technical Decisions

### 1. Text IDs Throughout ✅
All tables use `text('id').primaryKey()` for consistency with existing schemas.

### 2. JSONB for Task-Specific Data ✅
Flexible schema allows different fields per task type without NULL columns.

### 3. Separate Weight Tracking Table ✅
`weightRecords` separate from `tasks` enables historical charting and analysis.

### 4. Puppy Feeding Generator Batch Tracking ✅
`generationBatchId` groups related tasks, `puppyFeedingGenerations` tracks history.

### 5. Report Data Snapshots ✅
Complete data stored in JSONB prevents broken reports if source data changes.

### 6. Mating Comparison Caching ✅
Calculated comparison data cached in `comparisonData` JSONB for performance.

### 7. Export File Management ✅
Signed URLs with expiration, download tracking, file cleanup scheduling.

## 🔗 Integration Points

### With Animals Schema
- `tasks.animalId` → `animals.id`
- `weightRecords.animalId` → `animals.id`
- Puppy feeding links to puppies via `taskData.puppyFeedingData.puppyId`

### With Matings Schema
- `matingComparisons.mating1Id/2Id/3Id` → `matings.id`
- Progesterone data fetched from `animals.progesteroneReadings`

### With Litters Schema
- Puppy feeding generator queries `litters` and `puppies`
- Age calculation for feeding schedule (3-4 months = 3x/day, 4-6 months = 2x/day)

### With Users Schema
- All tables link to `users.id` for ownership
- `tasks.completedBy` tracks who completed tasks
- Export history per user

### Generate Puppy Feeding Tasks
```typescript
const batchId = generateId();
const tasksCreated = [];

// Create tasks
for (const puppy of eligiblePuppies) {
  tasksCreated.push({
    userId,
    type: 'puppy_feeding',
    isAutoGenerated: true,  // ⭐
    generatedBy: 'puppy_feeding_generator',  // ⭐
    generationBatchId: batchId,  // ⭐
    taskData: {
      puppyFeedingData: {  // ⭐
        litterId, puppyId, puppyName,
        puppyAge: '3-4 months',
        feedingsPerDay: 3,
        autoGenerated: true,
        generatedDate: today
      }
    }
  });
}

// Track generation
await db.insert(puppyFeedingGenerations).values({
  userId, batchId,
  tasksGenerated: tasksCreated.length,
  // ... other fields
});
```

// Generate report
